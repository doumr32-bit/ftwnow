## The Three HALT Paths

**v3.1 Feature** (tested in Test 4, validated in Test 7 logs):

When ⛔ HALT is triggered:

```
┌─ Resolve ─→ Fix problem, continue
├─ Alternative ─→ Use fallback plan, continue
└─ Explicit Skip ─→ Acknowledge risk, record to SKIPPED_GATES, continue
                    Phase 7: Review each skipped gate
```

Recorded in `halt-log.jsonl` with resolution field.

## The Five Splitting Strategies

**v3.1 Feature** (tested in Test 6, formalized in Test 1):

When file exceeds 400 lines:

1. **By Responsibility** — Split service into core logic + validators + crypto
2. **Extract Subcomponents** — Move React subcomponents to separate files
3. **Extract Utilities** — Move reusable logic to `lib/` or `utils/`
4. **Extract Types** — Move type definitions to `types.ts`
5. **Extract Constants** — Move hardcoded values to `constants.ts`

Validated by `refactor-cleaner.md` lines 42-50.
